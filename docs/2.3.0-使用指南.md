# MilvusPlus 2.3.0 使用指南

> 版本：**2.3.0**  
> SDK：`milvus-sdk-java **3.0.4**`  
> 基线：Java 8 · Spring Boot 2.7.x · Solon 2.7.x  
> 更新说明摘要见根目录 [CHANGELOG.md](../CHANGELOG.md)

本文面向「从 2.2.x 升级」以及「首次使用 2.3.0 新能力」的开发者，按场景说明怎么配、怎么写、怎么排错。

---

## 1. 2.3.0 相对 2.2.x 变了什么

| 类别 | 内容 |
|------|------|
| 依赖 | SDK `2.5.5` → **`3.0.4`**；去掉写死的 protobuf 3.24.0 |
| 修复 | addField、Gson、null key、and/or/not、radius/limit、缓存空 NPE 等 |
| Schema | `schema-mode` 自动演进集合结构 |
| 动态集合 | `forCollection` 分表/多租户 |
| 检索 | `vectorSearch` / `textSearch` / `page` / `options` / `filter` / `sqlWhere` |
| 写入 | `partial()` 部分更新（读改写，兼容 2.5+） |
| 运维 | `truncateCollection`、`ensureSchema` |
| 本地 | `docker/` 一键 Milvus；集成测试 `MilvusPlusIntegrationIT` |

**兼容提醒：**

- SDK 3.0.4 官方对齐 **Milvus 3.0.x**。本地 Docker 默认是 **2.5.14 standalone**（易部署），CRUD/检索/过滤已联调通过；3.0 专属能力需 3.0 服务端。
- Hybrid `ranker` 类型变为 `CreateCollectionReq.Function` 子类（`RRFRanker` / `WeightedRanker` 仍可用）。

## 2. 快速开始

### 2.1 依赖

**Spring Boot：**

```xml
<dependency>
    <groupId>org.dromara.milvus-plus</groupId>
    <artifactId>milvus-plus-boot-starter</artifactId>
    <version>2.3.0</version>
</dependency>
```

**Solon：**

```xml
<dependency>
    <groupId>org.dromara.milvus-plus</groupId>
    <artifactId>milvus-plus-solon-plugin</artifactId>
    <version>2.3.0</version>
</dependency>
```

**纯 core：**

```xml
<dependency>
    <groupId>org.dromara.milvus-plus</groupId>
    <artifactId>milvus-plus-core</artifactId>
    <version>2.3.0</version>
</dependency>
```

### 2.2 本地 Docker（推荐开发联调）

```bash
cd docker
docker compose up -d
curl -f http://localhost:9091/healthz   # 期望 OK
```

- SDK 地址：`http://localhost:19530`
- 镜像默认 DaoCloud 前缀（国内拉取更稳）；`milvus` 约 1.8GB，**首次可能要几分钟**
- 详见 [docker/README.md](../docker/README.md)

停止：

```bash
docker compose down
# 清数据：rm -rf volumes
```

### 2.3 配置（Spring / Solon）

```yaml
milvus:
  enable: true
  uri: http://localhost:19530
  # token / username / password 按环境选填
  packages:
    - com.example.entity          # 实体所在包（必填）
  open-log: true
  log-level: INFO
  banner: true

  # ===== 2.3.0 新增 =====
  # IGNORE    : 仅不存在时创建（兼容 2.2 默认行为）
  # VALIDATE  : 实体多出的字段 → 启动失败
  # AUTO_ADD  : 自动 add 缺失字段（推荐新项目）
  # RECREATE  : 删表重建（危险，需 enable-recreate=true）
  schema-mode: AUTO_ADD
  enable-recreate: false          # 生产必须 false
```

## 3. 实体与注解

### 3.1 基本实体

```java
@Data
@MilvusCollection(name = "face_collection", enableDynamicField = true)
@GenerateMilvusMapper   // Spring 下可生成 Mapper（若使用注解处理器）
public class Face {

    @MilvusField(name = "id", dataType = DataType.Int64,
            isPrimaryKey = true, autoID = false)
    private Long id;

    @MilvusField(name = "person_name", dataType = DataType.VarChar, maxLength = 64)
    private String personName;

    @MilvusField(name = "status", dataType = DataType.Int32)
    private Integer status;

    @MilvusField(name = "face_vector", dataType = DataType.FloatVector, dimension = 128)
    @MilvusIndex(
            indexType = IndexParam.IndexType.IVF_FLAT,
            metricType = IndexParam.MetricType.L2,
            indexName = "face_idx",
            extraParams = {@ExtraParam(key = "nlist", value = "100")}
    )
    private List<Float> faceVector;
}
```

### 3.2 2.3.0 注解增强

```java
@MilvusField(
    name = "age",
    dataType = DataType.Int32,
    nullable = true,              // 已有集合 AUTO_ADD 时强烈建议
    defaultValue = "0",           // 可选默认值（字符串形式）
    isClusteringKey = false       // 聚簇键
)
private Integer age;
```

**要点：**

- 服务端对「已有 collection 加字段」通常要求 **nullable 或 default**。
- `schema-mode=AUTO_ADD` 时，若字段既不可空又无默认值，Plus 会尽量强制 `nullable=true` 并打 warn。
- `DataType` 已随 SDK 支持更多类型（如 Int8Vector、Geometry 等），按业务选型即可。

### 3.3 BM25 / 文本检索字段

```java
@MilvusField(
    name = "content",
    dataType = DataType.VarChar,
    maxLength = 2048,
    enableAnalyzer = true,        // 自动生成 sparse + BM25 function
    enableMatch = true
)
private String content;
```

查询时推荐用语义 API（隐藏 `_sparse` 细节）：

```java
mapper.queryWrapper()
    .textSearch(Face::getContent, "关键词")
    .top(10)
    .query();
```

## 4. Mapper / Service 基本 CRUD

### 4.1 注入

```java
@Autowired
private FaceMilvusMapper faceMapper;   // 继承 BaseMilvusMapper<Face>

@Autowired
private MilvusService milvusService;   // 通用门面 + ICM/IVec/IAM
```

### 4.2 插入 / 按主键查删改

```java
// 插入
faceMapper.insert(face);
faceMapper.insert(list);

// 主键查询
faceMapper.getById(1L, 2L);
// 或
faceMapper.queryWrapper().byIds(1L, 2L);

// 按主键更新（整行语义；缺非空字段会回查补齐）
faceMapper.updateById(face);

// 部分更新（只改非 null 字段，内部读改写，兼容 2.5/3.x）
Face patch = new Face();
patch.setId(1L);
patch.setPersonName("新名字");
faceMapper.updateWrapper().partial().updateById(patch);

// 删除
faceMapper.removeById(1L);
```

### 4.3 条件查询（Lambda）

```java
// 简单 AND
faceMapper.queryWrapper()
    .eq(Face::getStatus, 1)
    .like(Face::getPersonName, "张")
    .query();

// 嵌套 OR（推荐 Consumer 写法）
// (status == 1) || (person_name == "A" && status == 0)
faceMapper.queryWrapper()
    .eq(Face::getStatus, 1)
    .or(w -> w.eq(Face::getPersonName, "A").eq(Face::getStatus, 0))
    .query();

// 嵌套 AND + 内层 OR
// status == 1 && ((type == 'A') || (type == 'B'))
faceMapper.queryWrapper()
    .eq(Face::getStatus, 1)
    .and(w -> w
        .eq(Face::getType, "A")
        .or(w2 -> w2.eq(Face::getType, "B")))
    .query();

// NOT
faceMapper.queryWrapper()
    .eq(Face::getStatus, 1)
    .eq(Face::getDeleted, true)
    .not()          // not (status == 1 && deleted == true)
    .query();
```

**逻辑约定（2.3.0 已修正）：**

| 写法 | 含义 |
|------|------|
| 连续 `eq/ne/...` | 彼此 **AND** |
| `and(nested)` | `(当前整组) && (nested 整组)`，nested 内默认 AND |
| `or(nested)` | `(当前整组) \|\| (nested 整组)`，nested 内默认 AND |
| `not()` | 对当前全部条件取反 |

## 5. 过滤：三种写法（由浅到深）

### 5.1 Lambda（主路径，类型安全）

见上一节。

### 5.2 类 SQL WHERE 子集（实验）

```java
faceMapper.queryWrapper()
    .sqlWhere("status = 1 AND person_name LIKE '%张%' OR type IN ('A','B')")
    .top(10)
    .query();
```

自动转换示例：

| SQL 子集 | Milvus 表达式 |
|----------|----------------|
| `status = 1` | `status == 1` |
| `AND` / `OR` / `NOT` | `&&` / `\|\|` / `not` |
| `LIKE '%x%'` | `like '%x%'` |
| `IN ('A','B')` | `in ['A','B']` |
| `IS NULL` / `IS NOT NULL` | `== null` / `!= null` |
| `<>` | `!=` |

**不支持**（会抛 `MilvusPlusException`）：`SELECT` / `JOIN` / `ORDER BY` / `BETWEEN` / 子查询等。  
这不是完整 MySQL，只覆盖日常过滤。

### 5.3 原生 Milvus 表达式

```java
faceMapper.queryWrapper()
    .filter("status == 1 && array_contains(tags, \"vip\")")
    // 或 .where("...")
    .query();
```

适合 JSON/Array 函数等 Lambda 尚未封装的能力。

## 6. 向量检索

### 6.1 语义化 API（推荐）

```java
List<Float> emb = ...;

faceMapper.queryWrapper()
    .vectorSearch(Face::getFaceVector, emb)
    .eq(Face::getStatus, 1)          // 标量过滤
    .top(10)                         // 等价 limit/topK
    .options(opt -> opt
        .metricType(IndexParam.MetricType.L2)
        .ignoreGrowing(true)
        .radius(0.8f)                // 范围检索
        .rangeFilter(0.1f))
    .query();
```

### 6.2 范围检索简写

```java
.vectorSearch(Face::getFaceVector, emb)
.range(0.1, 0.8)   // range_filter + radius
.top(20)
.query();
```

### 6.3 文本检索（BM25）

```java
.vectorSearch  // 稠密
// 或
.textSearch(Doc::getContent, "用户查询文本")
.top(10)
.query();
```

### 6.4 Hybrid（多路 + ranker）

```java
import io.milvus.v2.service.vector.request.ranker.RRFRanker;

faceMapper.queryWrapper()
    .hybrid(faceMapper.queryWrapper()
        .vectorSearch(Face::getFaceVector, emb).top(20))
    .hybrid(faceMapper.queryWrapper()
        .textSearch(Face::getContent, "关键词").top(20))
    .ranker(new RRFRanker(60))
    .top(10)
    .query();
```

### 6.5 分页注意

```java
// ✅ 标量分页
faceMapper.queryWrapper()
    .eq(Face::getStatus, 1)
    .page(1, 20);   // PageResult：total + records

// ❌ 向量 ANN 不要用 page()（会明确报错）
// 向量用 top/limit + 谨慎使用 offset
```

**无 filter 的 query** 会自动带默认 limit（避免服务端 `empty expression should be used with limit`）。

## 7. Schema 演进

### 7.1 启动行为

启动扫描 `packages` 下带 `@MilvusCollection` 的实体，按 `schema-mode` 处理：

| 模式 | 集合不存在 | 集合已存在 |
|------|------------|------------|
| `IGNORE` | 创建 | 只 load，不对比字段 |
| `VALIDATE` | 创建 | 实体多字段 → **失败** |
| `AUTO_ADD` | 创建 | **自动 add** 缺失字段 |
| `RECREATE` | 创建 | **删表重建**（需 `enable-recreate=true`） |

### 7.2 运行时 API

```java
// 默认 AUTO_ADD
milvusService.ensureSchema(Face.class);

// 指定策略
milvusService.ensureSchema(Face.class, SchemaMode.VALIDATE);

// 动态物理名 + 确保存在
milvusService.ensureSchema(Face.class, "face_tenant_01", SchemaMode.AUTO_ADD);
```

### 7.3 手动加字段

```java
// 已修复：会真正调用 addCollectionField
milvusService.addField("face_collection",
    AddFieldReq.builder()
        .fieldName("age")
        .dataType(DataType.Int32)
        .isNullable(true)
        .build());
```

## 8. 动态集合（分表 / 多租户）

```java
String coll = "face_" + tenantId;

// 先确保集合存在
milvusService.ensureSchema(Face.class, coll, SchemaMode.AUTO_ADD);

// 后续 CRUD 都打到该物理集合
faceMapper.forCollection(coll).insert(face);
faceMapper.forCollection(coll).queryWrapper()
    .eq(Face::getStatus, 1)
    .query();

// 或一次性绑定
BaseMilvusMapper.CollectionBoundMapper<Face> m =
    faceMapper.forCollection(coll);
m.insert(face);
m.getById(1L);

// Service 写法
milvusService.forCollection(Face.class, coll).insert(face);
```

逻辑实体注解名仍是「模板/默认名」；`forCollection` 覆盖物理 collection，并复用主键字段映射。

## 12. 从 2.2.x 升级检查清单

1. **依赖版本** 改为 `2.3.0`，确认传递依赖为 `milvus-sdk-java:3.0.4`。
2. **服务端**：生产尽量对齐 3.0.x；若暂用 2.5/2.6，先用本指南 + IT 回归核心路径。
3. **配置**：增加 `schema-mode`（新项目建议 `AUTO_ADD`）。
4. **代码**：
   - Hybrid ranker 确认仍用 `RRFRanker`/`WeightedRanker`。
   - 复杂条件改用 `or(w -> ...)` / `and(w -> ...)`，勿依赖旧错误语义。
   - 搜索优先 `top()` / `limit`，`metricType` 用枚举或 `options`。
5. **跑测试**（有本地 Docker 时）：


## 13. 完整示例（端到端）

```java
@Service
@RequiredArgsConstructor
public class FaceSearchService {

    private final FaceMilvusMapper faceMapper;
    private final MilvusService milvusService;

    @PostConstruct
    public void init() {
        milvusService.ensureSchema(Face.class);
    }

    public void save(Face face) {
        faceMapper.insert(face);
    }

    public List<Face> search(List<Float> emb, String keyword, int topK) {
        MilvusResp<List<MilvusResult<Face>>> resp = faceMapper.queryWrapper()
                .vectorSearch(Face::getFaceVector, emb)
                .eq(Face::getStatus, 1)
                .and(w -> {
                    if (keyword != null && !keyword.isEmpty()) {
                        w.like(Face::getPersonName, keyword);
                    }
                })
                .top(topK)
                .options(opt -> opt.metricType(IndexParam.MetricType.L2))
                .query();
        if (!resp.isSuccess() || resp.getData() == null) {
            return Collections.emptyList();
        }
        return resp.getData().stream()
                .map(MilvusResult::getEntity)
                .collect(Collectors.toList());
    }

    public PageResult<Face> pageActive(int pageNum, int pageSize) {
        return faceMapper.queryWrapper()
                .eq(Face::getStatus, 1)
                .page(pageNum, pageSize)
                .getData();
    }

    public void rename(Long id, String newName) {
        Face patch = new Face();
        patch.setId(id);
        patch.setPersonName(newName);
        faceMapper.updateWrapper().partial().updateById(patch);
    }
}
```

## 15. FAQ

**Q: sqlWhere 能写完整 SQL 吗？**  
A: 不能。只支持 WHERE 过滤子集。完整 SQL / JOIN 不是 Milvus 模型；复杂逻辑用 Lambda 或 `.filter` 原生表达式。

**Q: page 和 top 有什么区别？**  
A: `page` 给**标量查询**做 offset 分页并带 total；向量 ANN 用 `top/limit`。

**Q: partial 会改服务端未传字段吗？**  
A: 不会。Plus 会先读旧行，只覆盖你传入的非 null 字段再 upsert（兼容旧服务端）。

**Q: AUTO_ADD 会删字段吗？**  
A: 不会。只新增实体有、服务端没有的字段。删字段/改类型需人工迁移或 `RECREATE`（开发环境）。

**Q: Docker 里 minio 显示 unhealthy？**  
A: 部分镜像缺 `curl` 导致 healthcheck 失败，**不影响** Milvus `19530` 使用；以 `9091/healthz` 为准。

**Q: 集成测试在哪？**  
A: `milvus-plus-core/src/test/java/.../it/MilvusPlusIntegrationIT.java`  
无本地 Milvus 时会自动 skip；有服务则真实联调。
